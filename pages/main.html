Использовать препроцессор для создания страниц приложения "блог" - сверстанные страницы должны быть доступны на githubpages

Цель: В результате выполнения ДЗ вы создадите верстку для вашего личного блога с использование препроцессора SASS(/LESS)

В данном задании тренируются навыки:

  • верстки страниц
  • использование принципов DRY при верстке
  • использование препроцессоров
  • деплой страниц на сервис github pages

Необходимо:

  • создать новый репозиторий
  • настроить в нем линтеры, github actions
  • включить проверку линтером при PR
  • включить деплой приложения при PR
  • создать сайт-блог с минимум 4мя страницами (главная, обратная связь, список записей, страница записи)
  • создать пуллреквест с изменениями и
  • сбросить ссылку на PR и ссылку на задеплоенные страницы в част с преподавателем
Критерии оценки: Критерии оценки:
  • создан репозиторий, настроены линтеры - 1 балл
  • настроены actions, проверка линтерами и автодеплой - 1 балл
  • сверстаны 4 страницы с использованием препроцессора - 2 балла
  • есть дополнительные страницы - 1 балл
  • использована методология BEM - 2 балла

Оптимизировать сверстаные страницы по lighthouse и проверить на a11y

Цель: В процессе выполнения ДЗ вы узнаете по каким метрикам можно оценивать страницу

Тренируются навыки

  • использования инструментов оценки веб-страниц
  • оптимизация страниц под заданные параметры

Необходимо:

  • в репозитории прошлого задания сделать новую ветку
  • прогнать текущие страницы через lighthouse
  • сделать скриншоты оценок ДО
  • улучшить показати Performance, Accessibility, SEO, Best Practicies минимум до 95%
  • проверить страницы на https://validator.w3.org/ и исправить ошибки
  • сделать скриншоты оценок ПОСЛЕ
  • создать пуллреквест, в описание к пуллреквесту прикрепить скриншоты ДО/ПОСЛЕ (подписав)
  • ссылку на пуллреквест и на задеплоенные страницы сбросить в чат с преподавателем

Критерии оценки:

  • страница имеет все показатели не ниже 95% - 1 балл за страницу (не больше 4)
  • страница проходит проверку на https://validator.w3.org/ без ошибок - 1 балл за страницу (не более 2)
  • страница имеет все показатели 100% - 1 балл за страницу (не больше 4, если страница имеет все показатели 100 - это два балла - 1 по этому пункту и 1 по прошлому) Статус принято ставится от 4 баллов
